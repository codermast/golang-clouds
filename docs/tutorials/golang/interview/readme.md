## 高频考点

### 基础必考

- [ ] slice 和 array 的区别？slice 扩容机制？
- [ ] map 的底层实现？为什么是无序的？
- [ ] interface 的底层结构？空接口和非空接口？
- [ ] defer 的执行顺序？defer 遇到 panic？
- [ ] 值传递和引用传递？

### 并发必考

- [ ] Goroutine 和线程的区别？
- [ ] Channel 的底层实现？有缓冲和无缓冲的区别？
- [ ] GMP 调度模型？G、M、P 分别是什么？
- [ ] 如何控制 Goroutine 的并发数量？
- [ ] sync.Map 和普通 map 加锁的区别？

### 内存必考

- [ ] Go 的内存分配机制？TCMalloc？
- [ ] 什么是逃逸分析？如何避免逃逸？
- [ ] GC 三色标记法？写屏障？
- [ ] 如何排查内存泄漏？

### 项目必考

- [ ] Gin 中间件的实现原理？
- [ ] GORM 的 Hook 机制？
- [ ] 如何实现分布式锁？
- [ ] 如何处理高并发场景？

## 面试技巧

### 1. 回答框架

```
1. 先说结论（是什么）
2. 再说原理（为什么）
3. 举例说明（怎么用）
4. 延伸扩展（还知道什么）
```

### 2. 常见追问

面试官常见的追问方向：
- "底层实现是什么？"
- "有什么优缺点？"
- "在项目中怎么用的？"
- "遇到过什么问题？怎么解决的？"

### 3. 准备建议

- **基础要扎实**：slice、map、interface、channel 原理必须清楚
- **并发要深入**：GMP 调度是重中之重
- **结合项目**：准备 2-3 个能展示 Go 技能的项目经历
- **动手实践**：光看不练是不够的，要写代码验证
